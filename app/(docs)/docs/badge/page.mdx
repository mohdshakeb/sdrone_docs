# Badge

Badges are used to highlight status, category, or other metadata. They are non-interactive and provide at-a-glance information.

---

## Overview

The `Badge` component displays short labels with color-coded backgrounds. Use badges to:
- Indicate status (e.g., "Pending", "Approved", "Rejected")
- Categorize items (e.g., "New", "Featured")
- Show counts or tags

| Size | Height | Padding (H) | Font Size |

### Color Tokens

Badges use semantic status tokens that adapt to Light/Dark themes:

| Color | Subtle BG | Subtle FG | Intense BG | Intense FG |
|-------|-----------|-----------|------------|------------|
| `information` | `--bg-information-subtle` | `--fg-information-subtle` | `--bg-information-intense` | `--fg-information-intense` |
| `negative` | `--bg-negative-subtle` | `--fg-negative-subtle` | `--bg-negative-intense` | `--fg-negative-intense` |
| `neutral` | `--bg-neutral-subtle` | `--fg-neutral-subtle` | `--bg-neutral-intense` | `--fg-neutral-intense` |
| `notice` | `--bg-notice-subtle` | `--fg-notice-subtle` | `--bg-notice-intense` | `--fg-notice-intense` |

## Usage

```tsx
import Badge from '@/components/ui/Badge';

// Basic usage
<Badge>Default</Badge>

// With color and emphasis
<Badge color="positive" emphasis="intense">Approved</Badge>

// With size
<Badge color="negative" size="small">Error</Badge>
```
